# Development Log - 2026-06-18

## Implemented Changes

### 1. FLOSS ingestion integrated into core formatter pipeline

Implemented in `src/IngestionFormatters.jl`:

1. Added `:FLOSS` geometry in `get_campaign_geometry`:
	- tower heights: `[0.5, 1.0, 2.0, 4.0]`
	- roughness anchors: `z0m=0.001`, `z0h=0.0001`
2. Added FLOSS loader branch in `load_campaign_samples`:
	- reads both FLOSS-I and FLOSS-II NetCDF directories
3. Added FLOSS-specific readers:
	- `read_floss_netcdf`
	- `read_floss_tower_series`
	- `extract_time_series`
	- `finite_mean`
4. Implemented robust handling for mixed NetCDF layouts:
	- dynamic `u_*m` / `U_*m` variable discovery
	- 1D and 2D (`station x time`) reduction

## Interpretation Notes (Clearly Labeled)

These are interpretation-level statements based on current fitted models (not direct observational truth claims):

1. FLOSS exhibits a delayed instability threshold relative to CASES-99 (smaller `gamma_c_hopf`).
2. FLOSS crossing slope magnitude is shallower (`|dRe/dgamma|` lower), indicating less brittle local crossing behavior in this model.
3. FLOSS Hopf clock is faster (`beta` larger, period shorter) once crossing occurs.
