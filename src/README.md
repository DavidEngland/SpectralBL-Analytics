# `src/` Analysis Core: Spectral Boundary Layer & Attractor Identification Framework

This directory contains the core mathematical engine and numerical pipelines for analyzing structural stability transitions in **High-Latitude Boundary Layers (HLBL)**—specifically focused on tracking low-rank structural collapses under **Arctic Amplification (AA)** regimes.

The framework converts raw spatial-temporal velocity, shear, and temperature fields into reduced-order sparse differential models, identifying localized tipping points, structural bifurcations, and chaos transitions using data-driven dynamical systems theory.

## 📦 Module Blueprint Breakdown

### 1. `RegimeClassifier.jl`

* **Role:** Identifies structural stability phases from raw temporal sequences using a polar coordinate transformation proxy ($R, \Omega$).
* **Physics Matrix:** Distinguishes highly stratified, non-turbulent Arctic midnight regimes from highly active, multi-scale turbulent mixing regimes based on a variance threshold ($Var(R) \le \epsilon$).
* **Key Export:** `classify_regime`

### 2. `DelayEstimator.jl`

* **Role:** Optimizes time-lag parameters ($\tau$) for phase space trajectory reconstructions.
* **Mathematical Engine:** Utilizes linear auto-correlation down-crossings combined with an information-theoretic **Average Mutual Information (AMI)** optimization strategy to pinpoint the ideal geometric independence window.
* **Key Export:** `estimate_delay_details`, `compute_ami`

### 3. `WSINDyOperators.jl`

* **Role:** Prepares differential-integral mapping matrices using numerical finite differences and regularized projections.
* **Mathematical Engine:** Contrasts standard first-order Tikhonov smoothing regularizers ($L^T L$) against the **Weak-Form SINDy (WSINDy)** framework, projecting states through smooth, compact bell curves ($w(t) = \sin^2(\pi t) + 10^{-6}$) to transfer noise sensitivity off experimental data derivatives.
* **Key Export:** `compute_wsindy_operator`, `compute_tikhonov_operator`

### 4. `SpectralBL_Stage2.jl`

* **Role:** Acts as the primary stage 2 orchestrator, coordinating regime processing, lag estimation, and mathematical operator mapping over rolling time windows.
* **Mathematical Engine:** Executes **Takens' Embedding Theorem** to expand low-dimensional observations into multi-variate attractor matrices:

$$Z_i = \begin{bmatrix} \eta(t_i + 2\tau) & \eta(t_i + \tau) & \eta(t_i) \end{bmatrix}$$


* **Key Export:** `process_stage2_window`, `build_takens_state`

### 5. `SpectralDiagnostics.jl`

* **Role:** Implements a high-fidelity synthetic model data generator to mimic Arctic Amplification (AA) core plateau behaviors.
* **Physics Matrix:** Controls effective spatial embedding dimension scales ($D_{eff}$) and thermal inversion parameters to validate the tracking capability of downstream modules under strict boundary layer stratification.
* **Key Export:** `compute_arctic_record`, `summarize_records`

### 6. `Stage4_SparseRegression.jl`

* **Role:** Discovers symbolic, parsimonious ordinary differential equations governing the boundary layer system.
* **Mathematical Engine:** Projects state vectors into non-linear polynomial dictionaries ($\Theta(Z)$) and solves equations using a **Sequential Thresholded Least Squares (STLS)** loop to enforce $L_0$ sparse regularization, eliminating noise-fitting coefficients.
* **Key Export:** `stls_solve`, `build_library`

### 7. `Stage5_BifurcationAnalytics.jl`

* **Role:** Map and track stability branches across non-linear equations generated by SINDy regression models.
* **Mathematical Engine:** Couples an adaptive bisection predictor-secant extrapolator with a localized line-search Newton corrector method to trace steady states over varying parameter limits. Computes Jacobians and eigenvalues to flag critical **Hopf Bifurcation** thresholds.
* **Key Export:** `trace_continuation_branch`, `newton_equilibrium`

## ⚡ Mathematical & Performance Design Guidelines

To handle heavy temporal loops without memory degradation, the core source code conforms to strict performance specifications:

* **Zero-Allocation Views (`@view` / `@views`):** Array slicing allocations are completely bypassed using lightweight pointers to pre-allocated data matrix blocks.
* **Allocation-Free Math Sweeps:** Inner loop evaluation tasks (e.g., `evaluate_rhs` and `compute_jacobian` inside `Stage5`) completely avoid copying full matrices or computing intermediate arithmetic expressions on the heap, keeping loop computations fast and memory efficient.
* **Explicit Diagonal Inversion Bypassing:** Matrix operations like $Z^T W Z$ dodge raw diagonal expansions by leveraging fast row/column scaling broadcasts: `(Z' .* w') * Z`.
